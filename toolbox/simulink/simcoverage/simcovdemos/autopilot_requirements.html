Autopilot Functional Requirements

 

This document describes a set of requirements that apply to the altitude portion of an autopilot controller.  The example problem is explained in the paper

 

Aldrich, W.J., “Using Model Coverage Analysis to Improve the Controls Development Process,” AIAA Modeling and Simulation Technologies Conference, Monterey California, August 2002

 

 

Requirements

 

1. Altitude Climb Control

Requirement

Altitude climb control is entered whenever:

|Actual Altitude- Desired Altitude | > 1500

 

Units:

Actual Altitude – feet

Desired Altitude – feet

 

Explanation

When the autopilot is in altitude climb control mode the controller maintains a constant user selectable target climb rate.  The user selectable climb rate is always a positive number if the current altitude is above the target altitude the actual target climb rate is the negative of the user setting.

 

 

2. Altitude Hold

Requirement

Altitude hold mode is entered whenever

 

 |Actual Altitude- Desired Altitude | <

              30*Sample Period*(Pilot Climb Rate / 60)

 

Units:

Actual Altitude – feet

Desired Altitude – feet

Sample Period – seconds

Pilot Climb Rate – feet/minute

 

Explanation

The transition from climb mode to altitude hold is based on a threshold that is proportional to the Pilot Climb Rate.  At higher climb rates the transition occurs sooner to prevent excessive overshoot.

 

 

3. Autopilot Disable

Requirement

Altitude hold control and altitude climb control are disabled when autopilot enable is false

 

Explanation

Both control modes of the autopilot are can be disabled with a pilot setting.

 

 

4. Glide Slope Armed

Requirement

Glide-slope control is armed when its Glide Slope Enable and Glide Slope Signal are both true.

 

Units:

Glide Slope Enable – Logical

Glide Slope Signal – Logical

 

Explanation

ILS Glide Slope Control of altitude is only enabled when the pilot has enabled this mode and the Glide Slope Signal is true indicating the Glide Slope broadcast signal has been validated by the on board receiver.

 

 

5. Glide Slope Coupled

Requirement

Glide Slope control becomes coupled when the control is armed and (Glide Slope Angle Error > 0)  and Distance < 10000

 

Units:

Glide Slope Angle Error – Logical

Distance – feet

 

Explanation

When the autopilot is in altitude climb control mode the controller maintains a constant user selectable target climb rate.  The user selectable climb rate is always a positive number if the current altitude is above the target altitude the actual target climb rate is the negative of the user setting.
